## Task Details & Engineering Context

### BL-001: Incoming Message Telemetry Stream for In-Game Offers

- **Context**: Currently, incoming agent offer events can be triggered locally via `broadcastIncomingAgentEvent` or played back in mock mode. For live backend deployments, the mock client must listen for live incoming messages pushed from the backend operational agent runtime.
- **Integration Requirements**:
  1. Establish SSE (`EventSource`) or WebSocket listener at `/api/simulator/incoming-offers` or Pub/Sub subscription endpoint.
  2. Map incoming offer payloads to `broadcastIncomingAgentEvent()` in `simulatorBridge.ts`.
  3. Ensure the active `MockClientTab.tsx` viewport displays the dynamic offer pop-up overlay immediately upon receiving the incoming offer payload.
